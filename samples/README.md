# Samples

Real outputs from a real run of the auditor — committed so you can see what the tool produces without having to clone, install dependencies, and execute it yourself.

## Contents

### `input/`

- **`sample_messy_data.xlsx`** — A deliberately messy 30-row, 2-sheet Excel file (`Customers` + `Orders`) that exercises every detection category: mixed date / phone / currency formats, phantom duplicates separated only by whitespace and casing, placeholder floods (`Test`, `N/A`, `TBD`, `000-000-0000`), reference codes in name fields, free text in currency columns, and rows of pure missing data. Generated by `generate_sample.py` in the project root.

### `output/`

The three artifacts produced by a single audit run, all from the same input file:

- **`sample_messy_data_audit_report.html`** — Visual stakeholder report. Severity badges, format-distribution tables, null-completeness bars, and plain-English impact statements. Open it in a browser.
- **`sample_messy_data_audit_findings.xlsx`** — One row per issue. Frozen header, auto-filter, severity color-coding. Built for the person who has to actually do the cleanup.
- **`sample_messy_data_audit_report.pdf`** — Same substance as the HTML, formatted for email attachments and formal deliverables.

The run that produced these files reported **59 issues** — 23 High, 20 Medium, 16 Low.

### `output/screenshots/`

Screenshots of the HTML report rendered in a browser, for readers who want a quick preview without downloading the file.

## Reproducing

From the project root:

```
python audit.py --input samples/input/sample_messy_data.xlsx --output samples/output/
```
